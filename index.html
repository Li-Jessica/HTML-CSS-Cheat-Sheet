HTML Cheat Sheet

About HTML

The document type declaration <!DOCTYPE html> is required as the first line of an HTML document. The doctype declaration is an instruction to the browser about what type of document to expect and which version of HTML is being used, in this case it’s HTML5.

Basic Tags

Tag Name Description
<HTML> HTML Element Creates an HTML document. This should be added after the !DOCTYPE declaration. All content/structure for an HTML document should be contained between the opening and closing <HTML> tag.
<head> Head Element Contains general information about an HTML page that isn't displayed on the page itself. This information is called metadata and includes things like the title of the HTML document and links to stylesheets.
<body> Body Element This represents the content of an HTML document.
Note: there can only be 1 body element.
<title> Title Element Contains a text that defines the title of an HTML document. The title is displayed in the browser’s title bar or tab in which the HTML page is displayed. Note: This element can only be contained inside a document's <head> element.

Text Tags

Tag Name Description
<h1> - <h6> Heading Element The heading elements are ordered from the highest level <h1> (biggest) to the lowest level <h6> (smallest).
<strong> Bold Element Bolds text.
<em> Italicize Element Italicizes text.
<code> For code Used to define source code (usually in monospace).

Formatting

Tag Name Description
<p> Paragraph Element Contains and displays a block of text.
<span> Span Element An inline container for text and can be used to group text for styling purposes. However, as <span> is a generic container to separate pieces of text from a larger body of text, its use should be avoided if a more semantic element is available.
<br> Line Break Element Creates a line break in text and is especially useful where a division of text is required, like in a postal address.
Note: The line break element requires only an opening tag and must not have a closing tag.
<blockquote> Blockquote Element Puts content in a quote (indents text from both sides).
<div> Div Element Used as a container that divides an HTML document into sections. Can contain flow content such as headings, paragraphs, links, images, etc.

Links and its Attributes

Attribute Name Description
<a> Anchor Element The anchor element is used to create hyperlinks in an HTML document. The hyperlinks can point to other webpages, files on the same server, a location on the same page, or any other URL via the hyperlink reference attribute href.
href="../" href Attribute Determines the location the anchor element points to. Can be used to create text links or image links.
target Target Attribute Specifies where the hyperlink should be opened. E.g. target="_blank" will open the hyperlink in a new tab.
Examples
<a href="URL"> click text </a> Creates a hyperlink to an URL.
<a href="mailto:EMAIL_ADDRESS"> click text </a> Creates a hyperlink to an email address.
<a name="NAME"> click text </a> Creates a target location within a document.
<a href="#NAME"> click text </a> Creates a link to that target location.

Lists

Tag Name Description
<ul> Unordered List Element Used to create a list of items in no particular order. Each individual list item will have a bullet point by default.
<li> List Item Element Create list items inside ordered lists and unordered lists.
<ol> Ordered List Element Creates a list of items in sequential order (numbered lists).
<dl> Definition List Element Used to specify (start) the definition list. Commonly used to implement a glossary. Can put only <dt> and <dd> tags inside the <dl> tag.
<dt> Definition Term Element Preceds eacch defintion term. Used to specify the defintion terms/items in a definition list. This can be paragraphs, line breaks, images, links, lists, etc. within the <dd> tag. It can also use a <dt> tag within <details> and <figure> tags to represent a caption for content.
<dd> Defintion Element Precedes each definition. Used to describe an item within a list of defined terms in a definition list. This can be paragraphs, line breaks, images, links, lists, etc. within the <dd> tag.

Table Tags

Tag Name Description
<table> Table The wrapper element for all HTML tables.
<thead> Table Head The set of rows defining the column headers in a table.
<tbody> Table Body The set of rows containing actual table data.
<tr> Table Row The table row container. This is used to add rows to a table before adding table data <td> and table headings <th>
<td> Table Data The table data element (nested inside a table row <tr> element).
<tfoot> Table Foot The set of rows defining the footer in a table.

Table Attributes

Attribute Name Description
colspan Column Span Defines how many columns a <td> element should span.
rowspan Row Span Defines how many rows a <td> element should span.

Semantic HTML

Semantic HTML introduces meaning to the code we write. Before Semantic HTML the elements didn’t have any meaning as to what it does or what content goes in it. An element such as <div> was used as a general-purpose element to create things from headers to footers to articles. With Semantic HTML we were introduced to elements that tell developers and browsers exactly what it does and what content should go in it.

Element Placement

Semantic HTML introduces elements that can tell developers exactly what the element does or where it’s placed based on the name of that element. Some of these elements are <header>, <nav>, <main>, and <footer>. <header> describes the content at the top of the page <body>. It may include a logo, navigational links or a search bar. <nav> encapsulates the page’s navigational links. It is often placed inside the <header> or <footer>. <main> encapsulates the main content of a page between the header/navigation and the footer areas. <footer> includes the page’s footer content at the bottom of the <body>.

Embedding Media

Attribute Name Description
<video> Video Element Embeds a media player for video playback. The <src> attribute will contain the URL to the video. Adding the <controls> attribute will display video controls in the media player. May add "Video not supported" if the browser is unable to display the video.
<audio> Audio Element Embeds audio content. The audio type may also be specified (e.g. type="audio/mp3". Adding the attribute <controls> will display audio controls.
e.g. <video src="coding.mp4" controls>Video not supported</video>
<embed> Media Element Used to implement any type of media.
Note: <embeded> is deprecated. Other alternatives such as <video>, <audio>, and <img> should be used.
Other Useful Attributes
src The URL/source of the video/audio/embed content to be embed.
autoplay Boolean attribute. Automatically plays video/audio content.
loop Boolean attribute. Automatically loops video/audio content.

Figure (images) and Figure Captions

Attribute Name Description
<figure> Figure Element Used to encapsulate media such as an image, diagram, or code snippet.
<img> Imgage Element Embed images in documents.
<alt> Alt Attribute An <img> element can have alternative text via the <alt> attribute. The alternative text will be displayed if an image fails to render due to an incorrect URL, if the image format is not supported by the browser, if the image is blocked from being displayed, or if the image has not been received from the URL. The text will be read aloud if screen reading software is used and helps support visually impaired users by providing a text descriptor for the image content on a webpage.
<figcaption> Figure Caption Element used to describe the media encapsulated within the <figure> element. Developers will normally use <figcaption> within the <figure> element to group the media and description. This way, if a developer decides to change the position of the media, the description will follow along with it.

Text Elements

Attribute Name Description
<section> Section Element Defines elements in a document (e.g. chapters, headings, any other area of the documents with the same theme). Generally is used to define a theme for a webpage.
<article> Article Element Holds content that makes sense on its own such as articles, blogs, and comments. <article> is used to write independent content for the section theme.
<aside> Aside Element Used to mark additional information that can enhance another element but isn't required in order to understand the main content (e.g. bibliographies, endnotes, comments, pull quotes, editorial sidebars, additional info). Usually, this information would be in a sidebar or a location where it doesn't obstruct the main piece of content. Can be used alongside <article> or <section>.
